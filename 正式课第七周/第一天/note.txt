  1、DOS命令窗口  WIN键+R=>cmd
   [IP地址或者MAC地址]
    当我们的电脑连上网络后,路由器会给每一个电脑安排一个标识地址,这个地址就是我们的IP地址(例如:192.168.0.12 默认网关、子网掩码、DNS等)；不仅生成一个IP地址,而且结合电脑的出厂配置信息生成一个全世界唯一的标识码(MAC地址)；

    如果大家连接的是同一个WIFI(同一个路由器),默认网关和子网掩码是一样的,不同人之间的IP地址只有后面的数字不太一样,这种情况下说明大家在同一个局域网中,大家可以通过IP地址互相访问对方的电脑;(以后做移动端开发,在电脑上把项目开发完成,我们想在手机上看看效果,就可以让电脑和手机在同一个局域网下,我们在手机上就可以访问电脑上的项目了)
    命令：
    ipconfig
    ipconfig -all 查看本机的MAC地址(物理地址)

  [常用的一些小命令]
    cls 清屏
    exit 退出
    dir 查看当前目录下所有的文件

    cd 文件夹名称(进入到某一个文件夹)
    cd   / 返回根目录(一个斜杠没有点就是当前项目的根目录)
    cd  ./ 当前目录
    cd ../ 返回上一级目录

    D: 直接进入到D盘
    mkdir xxx 创建文件夹
    copy con xxx.xx 创建一个文件,接下来往文件中编写自己需要的内容即可,编写完成后需要保存退出的话:Ctrl+Z => Enter

  我们通过CD操作可以进入到某一个磁盘指定目录下,在这里运行DOS命令,但是我们可以更简单：
  直接先找到要操作的目录，在目录的空白处进行如下操作
         Shift+鼠标右键=>在此处打开命令窗口
  在目录的地址栏中,输出CMD也可以直接的进入DOS窗口,而且指定的就是当前文件目录

配置WB关于NODE代码的提示包
  File -> Settings -> Languages & Frameworks -> Node.js and NPM

  保证Node Interpreter 后面的框中有NODE的安装目录下的node.exe(把NODE集成到WB中) ，如果选择完成后，下面出现Version:7.0.1NODE的版本号

  [WB11及以上的]
    自带NODE的提示包,只需要点击ENABLE按钮即可

  [WB10及以下的]
    configure -> Archive file -> 把自己提前下载的提示包'node-v7.1.0.tar.gz'拷贝到NODE的安装目录下 -> 在Archive file中选择到这个提示包 -> config

检查是否安装配置成功
  File -> Settings -> Languages & Frameworks -> Javascript -> Libraries
  保证：HTML、HTML/ES5、ES6、Node..core 这四个是选中的,这样不管是前端还是NODE的提示都有了(以后只要没有提示的时候,就到这个位置把四个勾选上即可)

职业规划:
  从现在开始养成总结内容知识的能力,做一个自己的个人博客,去更新自己的文章内容,慢慢的让自己在这个圈子中有点名望(开放分享)
  =>我们自己做了一个个人博客,项目的源代码都在自己的电脑上呢,如果想让别人看的话,需要把这个项目发布出去,以后别人通过一个域名地址就可以看到我们的博客了

  如何发布自己的项目:
   1、买一台服务器(虚拟主机/云服务器 阿里云),把我们博客项目的资源文件上传到服务器上(FTP上传工具)；给服务器连上网，服务器就会有一个IP地址(外网IP:180.76.161.28)，以后用户(和我们的服务器不在同一个网下)通过这个外网IP就可以找到我们的服务器了。

   2、买域名(万网:https://wanwang.aliyun.com/)

   3、域名解析(DNS解析)
     把我们的域名和服务器关联到一起,因为解析域名的时候需要我们输入服务器的外网IP地址；从核心角度来讲，域名解析其实就是把域名和服务器的外网IP，相关联的这条记录，存放到DNS服务器上(DNS是全世界外网联盟管理的,不管是哪个国家的人注册域名进行解析，都会把相关的记录存放到DNS服务器上)

   4、在服务器上创建一个服务，让创建的服务负责接收客户端的请求，然后把客户需要的内容给客户，这样的话，我们的每一个项目都需要一个具体的服务来进行这些操作；=>一台服务器上如果要发布多个项目，那么需要有多个服务，如何区分这些服务呢？我们在创建服务的时候需要给每一个服务安排一个编号(端口号)，两个不同服务器的端口是不重复的；[在服务器上创建服务的方式很多：IIS、Apache、nginx、node...]
